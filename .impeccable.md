# Impeccable Design Context

## Design Context

### Users
Primary: Kristin (product designer at Viggle AI) and her collaborators, using the
hosted studio as a daily authoring instrument — building, tuning, and exporting
WebGL shader backgrounds that ship on real websites. Secondary — and still the
bar to design for: **strangers arriving cold at the studio URL** (or the public
GitHub repo). They should land, understand what it is, and produce something
beautiful within a minute — no README required.

### Product
A **hosted web app** (`studio/` — React + Vite + TypeScript, deploys on Vercel),
not a single-file artifact. The single-file idea lives on where it matters:
every shader **exports as one standalone self-contained HTML file** — the
artifact the studio produces is still view-source-able, even though the studio
itself is now a real product. The engines themselves remain live-editable JS
source strings; "edit the real source" is a feature, not an implementation
detail. The original single-file studio (`figma-shader-orb/index.html`) remains
in the repo as the reference implementation.

### Brand Personality
Precision instrument. Three words: **calibrated, dense, confident**.
The shaders are the expressive, colorful subject; the studio chrome around them
is lab equipment — an oscilloscope's bezel around a luminous signal. It should
evoke the feeling of being handed a professional tool, not a toy.
(The full visual system lives in `figma-shader-orb/DESIGN.md` — "The
Oscilloscope" — and governs both the legacy studio and the product.)

### Aesthetic Direction
- Monochrome chrome (near-black on near-white), JetBrains Mono UI text,
  uppercase micro-labels, thin borders, no decorative color in the panel —
  all color belongs to the shader canvas and swatches.
- Hero text overlay uses Satoshi (Fontshare) for website-preview realism.
- Signature shader palette recurs across presets: #f9b85f, #d9ccb0, #b6d9ff,
  #2a00ff on white.
- Light theme only (the canvas supplies darkness/color when needed).
- Anti-references: AI-slop dashboards (dark mode + neon glow, glassmorphism,
  gradient text), generic Tailwind card grids, rounded-everything SaaS chrome.

### Accessibility
WCAG AA: text contrast ≥ 4.5:1 (Dim Readout #6a6a6a is the lightest permitted
text grey), visible focus states everywhere (2px ink ring), keyboard
operability for all primary flows (⌘K command palette, ⌘Z history, slider
arrow-keys, focus-editable readouts), reduced-motion respected (shaders boot
paused; chrome transitions collapse).

### Design Principles
1. **Instrument, not toy** — density and precision over hand-holding; but every
   control must be labeled, legible, and predictable.
2. **Color lives on the canvas** — panel stays monochrome so the shader is the
   loudest thing on screen.
3. **The export is the artifact** — every shader leaves the studio as one
   self-contained, view-source-able HTML file; the product's honesty lives in
   what it ships.
4. **Every gesture reversible** — undo/redo, reset, revert; broken code edits
   revert themselves; experimentation must be free.
5. **First minute matters** — a cold visitor should reach "I made something"
   without instructions.
